## Objetivo do dia
- Configurar repositório **diario-de-estudos** e enviar primeiros arquivos.
- Aprender fluxo básico do Git: init, add, commit, push, pull.
- Criar estrutura de pastas por ano e iniciar o arquivo diário.

## O que eu fiz hoje
- Criei o repositório `diario-de-estudos` no GitHub.
- Inicializei git na pasta local (`git init`).
- Configurei user.name e user.email.
- Adicionei README.md e o primeiro arquivo de diário.
- Resolvi conflito inicial puxando mudanças do remoto (`git pull origin main --allow-unrelated-histories`).
- Executei `git push -u origin main` e confirmei que a branch `main` está rastreando `origin/main`.

## O que aprendi
- Fluxo mínimo Git: `git add .` → `git commit -m "mensagem"` → `git push`.
- Como ligar repositório local ao remoto: `git remote add origin <url>`.
- Como resolver quando remoto contém conteúdo: usar `git pull origin main --allow-unrelated-histories`.
- Branch principal padrão é `main`.

## Dificuldades / erros encontrados
- Mensagem: "Author identity unknown" — resolvido com:
  - `git config --global user.email "seu-email@gmail.com"`
  - `git config --global user.name "Alisson Vicente"`
- Push rejeitado porque remoto tinha README — resolvido com `git pull` e depois `git push`.

## Como resolvi
- Configurei identidade global do Git.
- Usei `git pull` para incorporar o README criado no GitHub.
- Reenviei com `git push -u origin main`.
